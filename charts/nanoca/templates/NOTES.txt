Security reminders:
{{- if .Values.secrets.create }}
  - secrets.create=true is rendering a chart-managed Secret. The values
    you supplied are stored in Helm release metadata (visible via
    `helm get values --all {{ .Release.Name }}` and `helm history`).
    Switch to secrets.existingSecret + a SealedSecret/ExternalSecret/sops
    workflow before going to production.
{{- end }}
{{- if not .Values.nanoca.remoteSigner.enabled }}
  - The CA private key is mounted into pods as ca.key. Consider enabling
    nanoca.remoteSigner so the CA private key never enters the cluster
    (HSM/KMS-backed signing oracle pattern). See values.yaml for details.
{{- end }}
{{- if not .Values.networkPolicy.enabled }}
  - networkPolicy.enabled=false. If your CNI supports NetworkPolicy
    (Calico, Cilium, etc.), enable it to restrict egress to known
    destinations (Fleet API, signing oracle, kube-dns, Apple/TPM root
    fetch). Default policy in values.yaml is a reasonable starting point.
{{- end }}
  - Secret values should be supplied via `--values <file>` (kept out of
    git), not `--set` (shell-history leak). When using existingSecret,
    pre-create it via SealedSecrets, ExternalSecrets, or sops.
